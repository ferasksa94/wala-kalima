# ولا كلمة 🎉

موقع لعبة حفلات عربية (RTL) يعمل بالكامل على المتصفح بدون أي خادم خلفي، ومناسب للنشر على **GitHub Pages**.

## المزايا

- عجلة اختيار عشوائية مع أنيميشن سلس.
- منع التكرار المباشر + خيار عدم التكرار حتى استخدام جميع العناصر.
- إدارة قائمة العناصر (إضافة، حذف، حذف الكل).
- استيراد/تصدير القائمة بصيغة JSON.
- نظام نقاط لفريقين (A/B) مع أزرار +1 / +5 / -1 وإعادة تعيين.
- مؤقت 120 ثانية (بدء/إيقاف/إعادة ضبط) مع رسالة انتهاء الوقت وصوت/اهتزاز.
- حفظ تلقائي في `localStorage` للأسماء، النقاط، والقائمة والإعدادات.
- يعمل **Offline after first load** باستخدام Service Worker.

## الملفات

- `index.html`
- `styles.css`
- `app.js`
- `sw.js`
- `README.md`

## التشغيل محليًا

1. افتح الملف `index.html` مباشرة في المتصفح، أو استخدم خادمًا محليًا بسيطًا.
2. (اختياري) عبر بايثون:

```bash
python3 -m http.server 8080
```

ثم افتح:

`http://localhost:8080`

## النشر على GitHub Pages (خطوات كاملة)

1. أنشئ مستودع جديد على GitHub (مثال: `wala-kalima`).
2. ارفع الملفات (`index.html`, `styles.css`, `app.js`, `sw.js`, `README.md`) إلى الفرع `main`.
3. في GitHub:
   - ادخل إلى **Settings** → **Pages**.
   - في **Build and deployment**:
     - Source = **Deploy from a branch**
     - Branch = **main**
     - Folder = **/ (root)**
4. اضغط **Save**.
5. انتظر دقيقة تقريبًا، ثم ستحصل على رابط الموقع (مثل):
   - `https://<username>.github.io/wala-kalima/`

## ملاحظات إنتاجية

- الموقع لا يحتاج build tools أو dependencies.
- يُنصح باستخدام HTTPS (GitHub Pages يفعله تلقائيًا).
- يفضل تحديث قيمة `CACHE_NAME` في `sw.js` عند أي إصدار جديد لضمان تحديث الملفات المخزنة.
